# Impeccable

## Design Context

### Users
Developers and technical PMs who chain HTTP APIs into runnable workflows on a
node canvas (Postman-meets-flowchart). They live in dark editors and terminals,
read monospace fluently, and judge tools by precision and speed.

### Brand Personality
Precise, calm, mechanical. A trading-desk instrument, not a marketing site.
Confidence comes from restraint — nothing glows unless it means something.

### Aesthetic Direction
- Dark only. Near-black `#070606`, cool grey ink `#d0dbda`, one orange accent
  `#ff5a19` (Ostium palette — defined in `src/app/globals.css`).
- Plus Jakarta Sans for prose/UI; JetBrains Mono for anything machine-readable
  (URLs, methods, template vars, code).
- Glass panels (`.glass*` utilities) are reserved for floating layers over the
  canvas: sidebar, top bar, nodes, modals. Never decorative elsewhere.
- HTTP method hues (`--m-get` green, `--m-post` orange, etc.) are semantic
  data colors, not accents.
- The dotted canvas is the product's landscape; UI that previews or teaches
  the canvas should reuse its dot texture and node vocabulary.

### Design Principles
1. Accent budget: orange marks the one primary action or selection per screen.
2. Structure is information — previews, chips, and connectors must depict real
   product behavior, never stock decoration.
3. Copy states outcomes in the user's vocabulary ("Every endpoint becomes a
   node"), buttons name the exact action ("Create workflow", not "Continue").
4. Mono for machine text, sans for human text — no exceptions.
5. Motion is one orchestrated entrance per surface; respect reduced motion.
